Release Notes - March 2026 (Version 28.1)

Dear Sweett Team,

In this release:

  • A Damage Protection service allows guests to replace the security deposit with a smaller one-time non-refundable fee in the Online Check-In.
  • Guests can now Resend invoice directly to their email address anytime after the stay.
  • A secure password manager is now built into SIBO — staff store and access third-party credentials with access controlled per individual user permissions.
  • Sales agents can now edit existing quote offers in place while the same link stays valid for the guest.
  • Nuki backup access code is now visible in SIBO in case Nuki's service is temporarily down
Damage Protection as Alternative to Deposit
Guests can now choose damage protection instead of a security deposit during Online Check-In — paying a small non-refundable fee instead of having a deposit held on their card, giving hosts an additional revenue stream while reducing checkout friction.
Resend Invoice
Guests can now resend their invoices directly to their email address anytime after the stay.
Secure Password Manager for Third-Party Systems
Staff can now store and access credentials for external systems (OTAs, payment providers, integrations) in a centralized password manager built into SIBO — with access controlled per individual user permissions, so each team member only sees what they're authorized to use.

Website & App

Guest Experience

  • Damage Protection as Alternative to Deposit
    Hosts can now offer damage protection as an alternative to a security deposit in the Online Check-In flow. Guests pay a small non-refundable fee instead of having a deposit held on their card, giving hosts an additional revenue stream while reducing checkout friction.
  • Download Generated Invoices
    After an invoice is generated, guests and users can download it directly from the Website or Guests App. This gives guests immediate access to their billing documents without needing to contact support.

SIBO

Security

  • Secure Password Manager for Third-Party Systems
    Staff can now store and retrieve credentials for external systems (OTAs, payment providers, integrations) in a centralized password manager built into SIBO. Access is controlled per individual user permissions, so each team member only sees the credentials they are authorized to use.

Direct Sales

  • Edit Existing Quote Offers
    Sales agents can now modify an existing quote (discount, dates, and other terms) directly in SIBO without creating a new offer link. The original link remains valid, so guests don't need to find or re-enter a new URL to complete their booking.

Stability

  • Persistent Admin Code for Nuki Locks
    A persistent admin access code for Nuki locks is now visible in SIBO to support emergency and crisis scenarios. This update also adds additional provider information and a direct link to the Nuki web portal for faster access when needed.

Enhancements & Bug Fixes

  • Enhancement Common Area Reports GateSIBO
    When creating a new report, SIBO now extends the "Review Existing Reports" gate to also include open reports for the same common area (building + common area). Staff can select an existing report and confirm — logging the acknowledgment and incrementing a counter on that report. If the issue isn't listed, the "Issue already reported, close report" option remains available without requiring a selection. This lightweight trail of acknowledgments forms a data basis for future issue prioritization.
  • Bug Fix Invoice Not Sent via Create & SendSIBO
    When clicking "Create & Send" for an invoice on Belgium-related bookings, the invoice was being created but not sent to the guest. This has been resolved so both actions complete correctly.
  • Bug Fix Agents Blocked from Ending ShiftSIBO
    Agents were incorrectly prevented from ending their shift even with no active cases. This occurred when a conversation was reassigned to another agent but the AI/human handler remained linked to the original agent, or when a new conversation was auto-assigned before it was escalated to a human. Both scenarios now correctly release the shift.
  • Bug Fix Paid Service Recorded TwiceSIBO
    When a guest paid for an optional add-on (e.g. Early Check-In or Late Check-Out) via a payment link, the service was recorded twice in the booking — once as paid and once as unpaid. This caused incorrect totals and potential errors in refund calculations. The duplicate entry has been fixed.
  • Bug Fix Incorrect Availability After OTA CancellationSIBO
    When a Booking.com or Airbnb booking was canceled or relocated while a SIBO overbooking existed on the same dates, the system incorrectly marked those dates as available — ignoring the active overbooking. The availability update now correctly reflects the actual database state after cancellation or relocation.
  • Bug Fix Airbnb Availability Check for Airbnb-only ListingsSIBO
    Airbnb availability checks were returning errors for listings whose properties are configured as Airbnb-only (not shown on the website), causing Airbnb to incorrectly mark those listings as unavailable. This has been resolved so availability is reported correctly for Airbnb-only properties.
  • Bug Fix Early/Late Confirmation Shows Wrong TimeSIBO
    When a late check-out request was approved at a different time than what was originally requested, the confirmation screen displayed the originally requested time rather than the actual approved time. This caused a confusing mismatch with the guest notification. The confirmation now correctly shows the approved time.
  • Bug Fix Misleading Promo Code MessageWebsite/App
    When a member discount outperformed a promo code, the system correctly applied the better discount but still displayed a green "Promo code applied" message. This has been updated to a clear message (e.g. "A better discount already applied") to accurately reflect what was used.
  • Bug Fix Booking Page Jumps on Slow NetworkWebsite/App
    On slow connections, the booking flow page would jump or shift as content loaded, making the experience unpredictable and difficult to use. Layout stability during progressive content loading has been improved.
  • Bug Fix Homepage Search Redirects to HomepageWebsite/App
    Clicking the search button on the homepage sometimes redirected users back to the homepage instead of the relevant city results page. This made it harder for visitors to find properties and could cause them to abandon their search. The redirect now consistently leads to the correct city page.
  • Bug Fix Quote & Membership Discounts Both AppliedWebsite/App
    When a quote discount was higher than a customer's membership discount, both were being shown and applied in the price breakdown, resulting in an incorrect total. The system now correctly applies only the better of the two discounts.
